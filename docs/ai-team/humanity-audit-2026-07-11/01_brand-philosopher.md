# 診断報告 — 「AI生成っぽさ」の構造的根本原因（brand-philosopher）

読んだ範囲: index.html（全体）、blog/index.html、blog/why-freed-time-fills-up/index.html、blog/margin-is-not-laziness/index.html、contact.html、chat.html、focus.html（構造のみ）、docs/ai-team/BRAND_PRINCIPLES.md。未検証: ブラウザでの実表示・EN版全文・focus.htmlのJS挙動。

## 1. 「AI生成っぽさ」の兆候 — どこに、なぜあるか

### 1-a. 同一骨格の11回反復【検証済み観察】
index.html のほぼ全セクションが `s-lbl（mono英語ラベル "// X"）→ s-h2（serif 2行見出し）→ s-desc（1段落・max-width 480px）→ 均等グリッド` という完全に同じ骨格。
- `#s-pain` index.html:1606-1608 / `#s-margin` :1619-1621 / `#s-service` :1653-1655 / `#s-cases` :1706-1708 / `#s-philosophy` :1822-1824 / `#s-faq` :1941-1943 / `#s-blog` :1999-2001
- 全セクションのパディングが同一トークン `clamp(80px, 9vw, 120px)`（:505）。`s-desc` は常に `max-width: 480px`（:529）
- セクション冒頭の1px罫線リビール（`.sec::before` :506-512）も全セクション共通

推論: 人間の編集者なら「このセクションだけラベルを省く」等の判断ムラが生まれる。ムラがゼロ＝生成システムの出力に見える最大の要因。
方向性: 少なくとも2セクションで骨格を崩す。例: `#s-philosophy` はラベル・descを捨てて引用（:1842-1845）から始める。`#s-about` は見出しの前に写真を置く。

### 1-b. 「3」の支配【検証済み観察】
3つ組がサイト全体で反復: h-chips 3枚（:1526-1540）、SIGNS 3項（:1610-1612）、margin-steps 3枚（:1631-1645）、entry-card 3枚（:1657-1671）、サービス 3枚（:1674-1700）、cases 3枚（:1710-1736）、scene 3幕（:1784-1803）、philosophy 3項（:1826-1840）、security-list 3項（:1858-1860）、blog 3枚（:2003-2020）、実績数字 3つ（:1920-1933）。**11箇所すべてが3**。
推論: "rule of three" はLLMの最頻出構成癖。2・4・5が混ざるだけで生成感は減る。
方向性: philosophy を「いま考えていること（増えたり減ったりします）」として4つ目を未完のまま置く、SIGNSに「4つ目はまだ名前をつけられていません」を足す等、数の非対称を1〜2箇所導入。

### 1-c. 見出しがすべて同じ修辞形【検証済み観察】
- 「余白は、探すものではなく、書き出して戻すもの。」（:1620）
- 「整えるのは業務ではなく、時間の使い道。」（:1654）
- 「効率化は目的ではなく、入り口だと考えています。」（:1823）
- 「余白は、怠けではない」（:2011）、「空白は怠けではなく、次の一手の置き場所」（:1839）

「XではなくY」の反転アフォリズムが見出しレベルで少なくとも5回。全見出しが「磨かれた格言」という同一声域にあり、平叙文・疑問・言い淀みがない。
推論: 1つなら知性、5つ並ぶと「格言ジェネレーター」。コピー品質ではなく声域の分散が無い問題。
方向性: 1〜2本を非格言形に。servicesは「やっていることを、そのまま並べます。」のような無防備な平叙文に。

### 1-d. 番号・monoラベルの偏在【検証済み観察】
`01/02/03` + monospace + letter-spacing のメタラベルが `h-chip-num`（:372-376）、`sign-num`（:617-621）、`margin-step-num`「01 / WRITE」（:1632）、`entry-kicker`「START 01」（:1658）、`sv-num`「01 / デザイン・発信」（:1677）、`phi-num`（:1827）、`security-list`（:1858）、`sst-tag`「scene 01」（:1785）、`source-num`（why-freed:422）と**9系統**。加えて「// SIGNS」等のコードコメント風プレフィックスと「CHOOSE YOUR FIRST STEP」「TOOLS WE THINK WITH」「A SMALL DIAGNOSTIC」（:1524, 1589, 1624）。
推論: 「英語monoラベル＋連番＋//」は2024〜25年のAI生成LPの最頻出シグネチャ。密度が高いと「システムが自分自身を注釈している」印象。
方向性: 半分に間引く。特に `entry-kicker`「START 01」と `margin-step-num`「01 / WRITE」は日本語と二重表記で削っても情報が減らない。

### 1-e. モーションの均質性【検証済み観察】
出現アニメーションはほぼ全て「opacity 0→1 + translateY(8〜20px) + `cubic-bezier(.16,1,.3,1)` + stagger .06〜.08s」の1種類（:186-194, :481-493, :527, :563-570, :607-616 等）。hover応答も「罫線色がサーモンに変わる」1種類（:571, :613, :637, :647, :676）。
推論: 統一自体はCalmに適合するが、全要素が同じ物理法則で動く世界には偶然性がない。

### 1-f. 具象の完全な不在【検証済み観察】
- サイト内の実写は `aimee-portrait.jpg` 1枚のみ、しかも `loading="lazy"` でページ下部（:1870）
- Visionの「写真」はCSS図形（`sst-moon` :762、`content:'AI'` の `sst-core` :806-816、`sst-particle` :821 等）
- 残りはツールロゴ壁（:1590-1601）と Tabler アイコン

推論: 抽象幾何＋ロゴ壁＋アイコンだけの画面は「AIが作ったプレースホルダー」の視覚言語。手・机・本・チェンマイの光といった固有の物体がゼロ。

### 1-g. ブランド規則の"完璧すぎる遵守"がボイラープレート化【検証済み観察】
- 同一記事内で「仮に30〜50%ほど短くなったとします」ヘッジ構文が2回ほぼ同文（why-freed:319, :362）
- 両記事が同一テンプレ: リード→目次→本論→同位置のinline-cta（文言完全同一）→反論処理→「抽象的な話で終えたくない」転換文→番号付き工夫→CTAボックス→参考文献
- FAQの回答とJSON-LD（index.html:97-155）が一言一句同じ

推論（診断の中核）: BRAND_PRINCIPLES.md の規則が例外なく・同じ言い回しで適用されている。揺らぎゼロの規則遵守は、それ自体がAIの指紋。原則が悪いのではなく執行が機械的。

## 3. Aimeeの視点が現れるべきなのに空白の箇所
- JAのヒーローに人間がいない: hero右カラムは業務時間チェッカー（:1543-1580）。EN版はヒーローに顔写真 `loading="eager"`（en/index.html:914 `hp-photo`）。→ JA heroにも小さな顔＋一行の署名的コピー。
- SIGNSの観察者が匿名: 「多くの経営者と話すなかで」（:1608）。ブログでは「平日5日間で20件近い打ち合わせ」（margin:326）と具体なのにLPでは主語が消える。→ SIGNSの1つに「私自身が3年前そうでした」を1文。
- philosophyの引用が借り物1つだけ: マルクス・アウレリウス（:1842-1845）はconsole（:2061）と重複。→ 「いま読み返している一節」＋日付の季節運用に。
- ブログの著者性: 記事末に顔・追記・時間の痕跡がない。→ 日付つき追記欄（P.S.）を許す。

## 4. 遊び心と Premium / Timeless / Calm は両立するか
両立する。原理3つ:
1. 遊びは「演じる」のではなく「発見される」— 読む速度で見つかる細部（欄外の一言、altテキスト、404、placeholder）に仕込む。
2. 遊びは具体性に宿る — 「読書家・哲学好き」（:1873）は属性ラベル。「いま読んでいる本: 『自省録』3周目」なら人間になる。
3. 声域の意図的な落差を1ページ1回 — 静かな面に1箇所だけ砕けた文を置く「点の遊び」。Calmはトーンでなく SN比の問題。

## 5. 「不完全さ・手の跡・偶然性」を導入すべき次元（優先順）
1. コピー（最優先・リスク最小）— 固有名詞・日付・失敗・言い淀みを表層に。ヘッジ構文の言い回しを毎回変える。
2. 画像 — sstの抽象CSS図形（:1751-1773)を実写3枚（作業机・手帳・チェンマイの窓）に置換。Ken Burns実装（:718-736）流用可。
3. レイアウト（欄外）— `--page-max: 1600px`（:202）の外側・記事の欄外に手書き風の傍注を1〜2個。
4. 構造 — 3つ組の一部を2や4に。セクション骨格の例外を2箇所。
5. モーション（最後・最小限）— 現状の統一はCalmに貢献。崩す優先度は低い。

## 最重要の根本原因トップ3
1. **単一テンプレートの無謬な反復** — 同一骨格×11、3つ組×11、格言見出し×5、番号ラベル×9系統。「単調」の正体は例外の完全な不在。
2. **人間味の"隠し要素への隔離"** — 人間の声はソース・console・コナミ・送信完了画面・fold下に封じられ、初見の表層は100%制度の声。JA heroに人間が不在（EN版には居る）。
3. **具象の不在＋規則遵守の機械化** — 実写1枚以外すべて抽象図形とロゴ。規則が一言一句レベルで反復され「正しさの均質さ」自体がAIの指紋。

一言でまとめ: **このサイトは"人間らしさの規則"を機械のように完璧に守っている**。直すべきは原則ではなく、執行に揺らぎ・固有・失敗を許すこと。変えるのは骨格の例外率と、Aimeeが表層に登場するタイミング。
