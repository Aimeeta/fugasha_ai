# 初見訪問者ウォークスルー — 「AIで作った感」の発生源調査（first-visit-researcher）

対象: index.html、blog/index.html、blog/why-freed-time-fills-up/index.html、contact.html、比較参照 en/index.html。実ブラウザ確認は未実施（静的読解のみ）。

## 1. 時系列ウォークスルー

### 1画面目（ヒーロー）— 「うまい。でも、誰？」
- 観察: H1「無駄をなくして、熱中を取り戻す。」（index.html:1517）、CTA2つ、チップ3枚、右に業務時間チェッカー（1544-1579）。**ファーストビューに人名・顔・一人称が一切ない。** 英語版は同じ位置に顔写真＋名前＋「Chiang Mai (UTC+7)」チップ（en/index.html:907-928、aimee-portrait-144.jpg は loading="eager"）。日本語版で aimee-portrait.jpg が出るのは About の1回だけ（:1870）、しかも lazy。
- 感情: コピーは上手い。チェッカーは触りたくなる（最初の感情の山）。だが「これは会社？一人？誰と話すことになるの？」が残る。
- 信頼が上がった点: 注記「※ ルーティン作業の30〜50%を任せられると仮定した概算です」（1578）。これは「文書への信頼」。

### 2画面目（TOOLS WE THINK WITH）— 最初のテンプレ感
- 観察: 英語ラベル（1589）＋ロゴピル10個（1590-1601）。
- 感情: AI系テンプレLPの定番中の定番。「あ、この型、見たことある」。日本の経営者には英語キッカーも届かない。

### 3〜7画面目（SIGNS → MARGIN MAP → SERVICES → WAYS OF WORKING）— 単調さの本体
- 観察: 構造が完全同型で反復。SIGNS=3項目（1609-1613）、MARGIN MAP=3ステップ01/02/03（1631-1645）、SERVICES=entry-card3枚＋bento3枚（1656-1701）、WAYS OF WORKING=カード3枚×各カード内もWHAT/PROCESS/FITの3行（1710-1737）。PHILOSOPHY 3項目、SECURITY 3項目、vision 3シーン。**「英語mono eyebrow＋serif2行見出し＋3つ組＋01/02/03」が7セクション連続。**
- 観察（言い回し）: H2レベルで「Xではなく、Y」構文が反復（1620, 1654, 1823、記事368）。
- 感情: 一文一文は上質なのにリズムが機械的等間隔で、3セクション目から読み飛ばしモード。「AIが書いたっぽい」引き金は (a)全部3つ組番号付き (b)対句構文の頻度 (c)固有名詞・実例・失敗ゼロ。WHAT/PROCESS/FITはフレームワーク臭が強く退屈の底。
- 一人称の初出は「多くの経営者と話すなかで」（1608）だが主語が消えている。

### 8〜9画面目（vision、280vhスクロールストーリー）— 見せ場に人がいない
- 観察: `.sst-outer { height: 280vh }`（:695）。「深夜2時。また、同じ作業。」（1786）→ ビジュアルは抽象CSSアニメ（1751-1773）。
- 感情: 本サイトで一番野心的な演出。だが「誰の深夜2時なのか」が語られず匿名の情景＝ストックコピーに読める。3画面分拘束される割に情報が抽象論のみで、離脱の最有力ポイント。

### 10画面目（About）— ようやく「人がいる」。最初で最後
- 観察: 顔写真（1870）、「以前の私も〜」（1878）、「空いた時間にふと窓の外を見て、『あれ、今日まだ明るい』と気づいた」（1879）。ph-badgeフリップ「読書家・哲学好き」（1871-1874）、about-miniスライド「チェンマイを拠点に」（1898）、実績数字 144本/6年/50社（1919-1934）。
- 感情: **「この裏に人がいる」と確信するのはここが初めて。実質9〜10スクロール目**（280vh含む）。「あれ、今日まだ明るい」はサイト全体で最も体温の高い一文。チェンマイ・読書家・三言語も魅力的なのにスライダーの中で小さく流れる。忙しい経営者がここまで読む前提は成立しにくい。

### FAQ / Blogプレビュー / CTA
- FAQ回答「うまくいった手順は、私がいなくても回る形にしてお渡しします」（1964）— 声がある良い箇所。CTA「急かしません。〜それで十分です。」（2031）も良い。
- 隠れた人間味: ソースコメント（1469-1474）、consoleイースターエッグ（2059-2062）、コナミコマンドのトースト（1479-1482）。**最も人間味のあるテキストが初見訪問者の99%には見えない場所にある。**

## 2. blogと記事の体温
- 一覧: 全カード同一フォーマット、著者表記・顔なし。日付が2026.07.01〜07.10の10日間に8本集中。「10日で8本、全部同じ長さ・同じトーン」は**AI量産ブログの視覚的シグネチャ**に見える。
- 記事（why-freed-time-fills-up）: 反論への応答（349-355）や「うまくいかなかったこと」（374）などAIが書かない粘りがある。**しかし**「私が実際にやってみたこと」（357）直後の段落が「仮に〜30〜50%ほど短くなったとします」（362）。リード直後の体験談も「たとえばの話ですが」（319）。「実際にやってみた」直後に「仮に」が来ると実体験か思考実験か分からなくなり体温が一気に下がる。正直さがかえって「本当はやってないのでは」の疑念を生む。面談枠の失敗談（333）は具体的で良い。
- 参考文献: Wikipedia 2件（421-443）だが本文にどちらの語も登場しない。本文と接続しない参考文献は「AIが機械的に付けた出典」に見える典型。
- 著者情報はメタ行の「Aimee — 風雅舎代表」1スパンのみ（303）。写真付き著者ボックスなし。

## 3. contact.html — 問い合わせ直前の気持ち
- 良い点: 会話風Q&A、「まだ送信していません」（354）、「営業目的では使いません」（320）、placeholder「『うちはここが一番きつい』でも〜なんでも。」（339）。
- 不安: **送信完了画面（390）まで受け手の顔も名前も出ない。** バブルアバターは「風」の一文字（454）。
- 実害級の発見: 送信失敗時「直接メールにてご連絡ください」（732）なのに、**contact.htmlのどこにもメールアドレスが記載されていない**（hello@fugasha.jp は en/index.html:903 にのみ明記）。失敗した訪問者は行き止まり。

## 4. 信頼の源泉の分析
現状の信頼はほぼすべて「文書への信頼」（仮定明記・NDA/セキュリティFAQ・送信前確認・a11yの丁寧さ）。「人への信頼」（顔・名前・拠点・失敗談・実名の仕事）はAboutと送信完了画面に隔離。「AIで作った感」の正体は文章力不足ではなく、**人の証拠の配置（ほぼ最後）と構造の等間隔な反復**。

## 5. 人間味が最も不足している場所トップ5（初見インパクト順）
1. **日本語版ヒーローに「誰」がいない**（index.html:1514-1585）。ENのhp-photoパターン（144pxポートレート＋名前＋一行）をJAヒーローかその直下に移植。既存アセットで追加コストゼロ。
2. **「英語キッカー＋3つ組＋01/02/03」が7セクション連続**。少なくとも2セクションでリズムを崩す — SIGNSを「実際に聞いた言葉」の引用体に、WHAT/PROCESS/FITを散文1段落に。数を2や4にするだけでも減る。
3. **ブログの書き手不在**: 一覧に著者ゼロ、日付が10日に8本集中、体験談の「仮に〜とします」、本文と接続しない参考文献。→ 記事末に写真付き著者ボックス、「実際にやってみたこと」は仮定表記をやめるか「正確な時間は測っていませんが」に。参考文献は本文中で触れるか外す。
4. **vision（280vh）の匿名の情景**。scene 01をAimee自身の一人称に書き換える（Aboutの「あれ、今日まだ明るい」の原体験はまさにこのシーンの実話のはず。ここで語ればAboutとの呼応も生まれる）。
5. **contact.htmlに受け手の顔がない＋失敗時の連絡先が空白**（454, 732）。Step1のバブルアバターをAimeeの顔写真に、「読むのは代表のAimee本人です／hello@fugasha.jp」を明記。732のエラーメッセージにアドレス直書き（人間味以前の正しさの問題）。

補足: サイトで一番Aimeeらしいテキスト（ソースコメント1469-1474、console 2059-2062）が不可視領域にある。あの声のトーンをそのまま本文側（特にヒーロー付近とcontact）へ数行引き上げるのが最小工数で「AIで作った感」を崩す近道。
